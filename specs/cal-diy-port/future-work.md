# Cal.diy Port Future Work

These areas are classified but deferred from Solo v1 unless a later approved implementation slice pulls them forward.

## Deferred Connectors

- CRM: Salesforce, HubSpot, Pipedrive, Close, Zoho, Attio, Basecamp and related packages.
- Analytics: GA4, GTM, PostHog, Plausible, Umami, Matomo, Meta Pixel, Twipla, Databuddy.
- Payments: Stripe, PayPal, BTCPay, HitPay, Alby and payment E2E flows.
- Extra calendar/video providers: Apple Calendar, CalDAV, Exchange, Daily, Whereby, Webex, Jitsi, Skype, FaceTime, Tandem, and similar non-selected providers.
- Automation/site/misc integrations: Zapier, Make, n8n, WordPress, Raycast, Pipedream and similar packages.

## Deferred Product Areas

- Public web booking pages.
- Embeds and Booker embed packages.
- Marketplace breadth beyond selected workforce connectors.
- Team/organization depth beyond Solo compatibility abstractions.
- Billing/payment upgrade flows except Solo plan eligibility display.
- Advanced analytics/reporting beyond booking audit and operational reporting required by Solo scheduling.

## Pull-Forward Rule

A deferred area can move into scope only if a concrete implementation task states which included feature depends on it, which Cal.diy source paths/tests become required, which Nerova target files and tests will own it, and which existing v1 scope item is displaced or expanded.
